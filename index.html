Apache JMeter

JMeter is not a browser

  • Web - HTTP, HTTPS (Java, NodeJS, PHP, ASP.NET, …)
  • SOAP / REST Webservices
  • FTP
  • Database via JDBC
  • LDAP
  • Message-oriented middleware (MOM) via JMS
  • Mail - SMTP(S), POP3(S) and IMAP(S)
  • TCP
  • Java Objects

jmeter works at protocol level. As far as web-services and remote services are concerned, JMeter looks like a browser (or rather, multiple browsers); however JMeter does not perform all the actions supported by browsers. In particular, JMeter does not execute the Javascript found in HTML pages. Nor does it render the HTML pages as a browser does (it's possible to view the response as HTML etc., but the timings are not included in any samples, and only one sample in one thread is ever displayed at a time).

  • Full featured Test IDE that allows fast Test Plan recording (from Browsers or native applications), building and debugging.
  • Command-line mode (Non GUI / headless mode) to load test from any Java compatible OS (Linux, Windows, Mac OSX, …)
  • A complete and ready to present dynamic HTML report
  • Highly Extensible core:
  • JMeter may be used for Performance, Load, and Stress testing. Jmeter also can be used for API testing why custom code? because ajax, weird jms configuration

Apache JMeter

Features

  • Multithreaded
  • Distributed
  • Integrated recording
  • Error handling
  • Timing & Scheduling
  • Extensible
  • Scriptable - command line
  • extensive report capability
Why JMeter? Why not build something?
I see JMeter as a pluggable thread-management platform. The core feature is its ability to spawn and synchronize threads, including the recording of data they generate. JMeter makes it easy to multithread requests, collaborate across multiple machines, record results in combination across all testing nodes, and it handles errors gracefully. Also, as you are about to see, getting started with JMeter is not that hard.

Getting Started

  • Download JMeter
  • Extract folder
  • Run bin/jmeter.sh or bin/jmeter.bat

Download (binary) of JMeter is a ZIP or TAR.GZ containing a single folder.

Simple Test

  • Thread Group (test case)
  • Sampler (step)
  • Listener (reporter)

Basic HTTP Test

  • Thread Group
  • HTTP Request Sampler
    • Timer
    • Pre sampler action
    • Post sampler action
  • Summary Report Listener

Basic JDBC Test

  • JDBC Connection Configuration
  • JDBC driver jar
  • Thread Group
  • JDBC Request Sampler
  • Summary Report Listener
Before running a JDBC test you'll need to install the JDBC Driver JAR. To do this, simply copy the JAR into the /lib/ folder of JMeter. Alternatively, you may modify the user.classpath in jmeter.properties to include the location of your JDBC Driver JAR.

Correlation

Correlation is the most important aspect of scripting on secure site (saml).

It generally includes fetching dynamic data from preceding requests/calls and posting it to the subsequent requests

  • regular expression
  • xpath
  • json
  • JDBC
  • custom beanshell/groovy/Java
if we just playback this script, the test will fail even for a single user. This is because of the authentication mechanism used. When we login to a website, session variables are dynamically created. These session variables are passed to the subsequent requests and help validation and authentication of the actions performed. So, one cannot just record and playback the requests having these variables. Here, we need to correlate the web requests with the dynamic variables. And for correlation, we need to use the "Regular Expression Extractor" which makes use of regular expressions.

Multiple nodes

source

Part of the power of JMeter is the ability to not only spawn multiple threads, but to synchronize threads across multiple machines in a network. This can be a great candidate for temporary nodes on EC2. Start using Run > Remote Start All, clicking double green arrows, or CTRL+SHIFT+R.
  • Configure nodes (Same Java & JMeter versions)
  • Start servers (/bin/jmeter-server)
  • List servers in /bin/jmeter.properties (remote_hosts)
  • Start JMeter Client from GUI

Run from command line

jmeter -n -t script.jmx -R 10.10.0.1,10.10.0.2,hostname3
-n means no GUI, -t specifies JMX script, -R provides remote servers

Extending JMeter: Customization

  • <component_name>BeanInfo.java

JMeter can be extended by creating plugins in Java and putting the JAR into the lib/ext folder. Here is an example of a simple JDBC sampler I created for testing purposes. We discovered that this was necessary to isolate DB performance from network performance.

SimplePlugin.java


							package devmamak.jmeter.simplePlugin;
							
							import org.apache.jmeter.samplers.AbstractSampler;
							import org.apache.jmeter.samplers.Entry;
							import org.apache.jmeter.samplers.SampleResult;
							import org.apache.jmeter.testbeans.TestBean;
							
							public class SimplePlugin extends AbstractSampler implements TestBean {
							
								private static final long serialVersionUID = 1L;
								
								private String name;
								private String say;
							
								public String getName() {
									return name;
								}
							
								public void setName(String name) {
									this.name = name;
								}
							
								public String getSay() {
									return say;
								}
							
								public void setSay(String say) {
									this.say = say;
								}
							
								@Override
								public SampleResult sample(Entry e) {
									SampleResult res = new SampleResult();
									res.setSampleLabel(getName());
									res.sampleStart();
									
									res.setResponseData("Hello Dev@Mamak, " 
									+ this.name 
									+ " says that \"" 
									+ this.say + "\"", null);
									res.setDataType(SampleResult.TEXT);
									res.sampleEnd();
									res.setResponseCodeOK();
									res.setResponseMessage("yeah, dev@mamak");
									res.setSuccessful(true);
									return res;
								}
							
							}

SimplePluginBeanInfo.java


								package devmamak.jmeter.simplePlugin;
								
								import java.beans.PropertyDescriptor;
								
								import org.apache.jmeter.testbeans.BeanInfoSupport;
								import org.apache.jmeter.testbeans.gui.TypeEditor;
								
								public class SimplePluginBeanInfo extends BeanInfoSupport {
								
									private PropertyDescriptor getprop(String name, Object deflt) {
										PropertyDescriptor p = property(name);
										p.setValue(DEFAULT, deflt);
										p.setValue(NOT_UNDEFINED, Boolean.TRUE);
										return p;
									}
								
									public SimplePluginBeanInfo() {
										super(SimplePlugin.class);
										getprop("name","guys");
										property("say", TypeEditor.TextAreaEditor);
									}
								
								}
